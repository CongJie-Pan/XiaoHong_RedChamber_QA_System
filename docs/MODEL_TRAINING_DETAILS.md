# 小紅問答系統：模型訓練與復現細節 (Model Training & Reproducibility)

本文件旨在提供系統核心微調模型 (XiaoHong-v1) 之訓練資料格式、模型超參數設定以及系統之真實對話範例，供相關研究領域之學者與開發者參考及復現。

## 一、模型訓練之「問題－背景知識－答案」三元組格式範例

### 1. 格式範例概要

| 類型 | 角色 | 內容摘要（節錄自真實資料集） |
| :--- | :--- | :--- |
| **System Prompt** | `system` | 你是一位專業的古典文學與知識問答助手。遇到需要深度分析與邏輯推演的問題，請先在 `<think>` 標籤內進行思考；若是簡單的事實擷取或問候，請直接給出答案，不需思考過程。 |
| **（一）標準 RAG 三元組** | `user` | 【參考資料】……十三經形成歷史……<br>【問題】請說明「五、十二經到十三經的演進」。 |
| | `assistant` | `<think></think>`（空白，屬簡單事實題）→ 直接回答：「漢代立五經→唐代開成石經增為十二經→北宋科舉與徽宗石經提升《孟子》確立十三經。」 |
| **（二）深度推理（含 CoT）** | `user` | 【參考資料】……桃葉渡、王獻之典故……<br>【問題】「桃枝桃葉總分離」是「字面引用」還是「反其意而用」？ |
| | `assistant` | `<think>` 1. 檢索原典，王獻之「渡江不用楫」強調歡聚可能。2. 分析「總分離」永訣語意與原典差異。3. 確認屬反其意而用。`</think>` → 結論：逆用典故，以哀轉喜，完成「地理符號→命運詛咒」的轉換。 |
| **（三）負樣本（拒絕回答）** | `user` | 【參考資料】……（與問題不相關之文獻）……<br>【問題】請分析「蛟綃」典故。 |
| | `assistant` | `<think></think>` → 「所提供的參考資料未包含足夠資訊，為維護準確性，無法在缺乏依據下作推測，還望理解。」 |

### 2. JSON 格式範例

```json
{
  "messages": [
    {
      "role": "system",
      "content": "你是一位專業的古典文學與知識問答助手。遇到需要深度分析與邏輯推演的問題，請先在 <think> 標籤內進行思考；若是簡單的事實擷取或問候，請直接給出答案，不需思考過程。"
    },
    {
      "role": "user",
      "content": "根據以下段落回答問題，若段落資訊不足以回答，請說明無法回答。\n\n【參考資料】\n從六經、五經到七經……十三經之名至此確立；南宋紹熙間黃唐合刻《十三經註疏》，成為後世標準版本。\n\n【問題】\n在「五經轉為十三經之關係」這個主題下，請說明「五、十二經到十三經的演進」。"
    },
    {
      "role": "assistant",
      "content": "<think>\n</think>\n由五而十二，關鍵在唐代官學擴張；由十二而十三，則是北宋科舉與徽宗石經共同抬升《孟子》的結果，並非朱熹《四書》流行後才發生。"
    }
  ],
  "sample_id": "c1a79991dfac504a58ca373c88d0bd55",
  "source_line": 6434
}
```

## 二、模型訓練之超參數與環境設置概要表

| 模型微調超參數設定 | 參數具體數值 |
| :--- | :--- |
| **基礎模型 (Base Model)** | Qwen/Qwen3-8B |
| **訓練伺服器** | AWS EC2 `g6e.xlarge` (1 × NVIDIA A10G 24GB)<br>AMI: Deep Learning Base AMI with Single CUDA (Ubuntu 22.04) - 64-bit (x86) |
| **QLoRA 參數**<br>(Rank / Alpha / Dropout / Target Modules) | `64` / `128` / `0.05` / `All Linear` |
| **優化器與調度器** | `paged_adamw_8bit` / `cosine_with_min_lr` |
| **學習率 (Learning Rate) / Epochs** | `2e-5` / `2.0` |
| **批次與梯度累積** (Batch / Grad-Acc) | `1` (per device) / `8` |
